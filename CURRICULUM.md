### 📖 MODULE 1.1: INTRODUCTION TO HTML5 (Beginner)

#### 🔤 Topic: What is HTML?

| Field | Details |
|-------|---------|
| **Difficulty** | ⭐ Beginner |
| **Estimated Time** | 30 minutes |
| **Prerequisites** | None |
| **Learning Objectives** | Understand what HTML is, its purpose, and how it fits in web development |

**📖 Definition:**  
HTML (HyperText Markup Language) is the standard language for creating web pages. It tells the browser how to structure and display content. "HyperText" means text with links, "Markup" means using tags to label content, and "Language" means it has its own rules.

**🧠 Roman English Explanation:**  
HTML ek aisi language hai jo browser ko batati hai ki webpage par kya dikhana hai. Jaise aap MS Word mein text likhte ho aur usme bold, italic lagate ho, waise hi HTML tags use karta hai content ko style karne ke liye. Lekin HTML sirf structure banata hai — styling CSS karta hai, aur behavior JavaScript.

**🔬 Technical Explanation:**  
HTML is a markup language that uses tags enclosed in angle brackets (`<tag>`) to define elements. Browsers parse HTML and render a Document Object Model (DOM) tree. HTML5 is the latest major version (released 2014, continuously updated). It is maintained by the WHATWG (Web Hypertext Application Technology Working Group).

**👶 Simple Explanation (12-year-old level):**  
HTML is like the skeleton of a website. Just like your bones give your body shape, HTML gives shape to a webpage. It decides where the heading goes, where the paragraph goes, where the image goes, etc.

**🎯 Analogy:**  
Think of HTML as the blueprint of a house. The blueprint tells the builder: "Here is the kitchen, here is the bedroom, here is the bathroom." Similarly, HTML tells the browser: "Here is a heading, here is a paragraph, here is an image."

**🌍 Real World Example:**  
When you open google.com, the Google logo, search box, and buttons are all placed using HTML. Facebook's news feed, profile pictures, and posts are all HTML elements arranged in a structure.

**💻 Code Syntax:**
```html
<!DOCTYPE html>
<html>
<head>
  <title>My First Page</title>
</head>
<body>
  <h1>Hello World!</h1>
  <p>This is my first webpage.</p>
</body>
</html>
```

**📜 Rules:**
1. HTML tags usually come in pairs: `<tag>` opening and `</tag>` closing
2. Some tags are self-closing: `<br>` or `<img>`
3. Tags can be nested inside each other
4. Attribute values should be in quotes
5. HTML is case-insensitive but lowercase is standard

**📌 Important Notes:**
- HTML is NOT a programming language — it is a markup language
- Every HTML document must start with `<!DOCTYPE html>`
- The `<head>` contains metadata, the `<body>` contains visible content

#### 🔤 Topic: HTML Document Structure

| Field | Details |
|-------|---------|
| **Difficulty** | ⭐ Beginner |
| **Estimated Time** | 45 minutes |
| **Prerequisites** | What is HTML |

**📖 Definition:**  
The HTML document structure is the standard skeleton that every HTML page follows. It includes the doctype declaration, html tag, head section, and body section.

**🧠 Roman English Explanation:**  
Har HTML page ka ek fixed structure hota hai. Pehle `<!DOCTYPE html>` likhte hain jo browser ko batata hai ki yeh HTML5 hai. Phir `<html>` tag ke andar do main parts hote hain: `<head>` (jo page ke baare mein information rakhta hai, jaise title) aur `<body>` (jo actually browser mein dikhta hai).

**🔬 Technical Explanation:**  
The HTML document structure follows a strict tree hierarchy. The `<!DOCTYPE html>` declaration must be the very first line. The `<html>` element is the root element. `<head>` contains meta-information, links to stylesheets, scripts, and the page title. `<body>` contains all visible content. This structure is enforced by the HTML parser in all browsers.

**👶 Simple Explanation:**  
Think of an HTML document like a school notebook. The cover is `<!DOCTYPE html>`. The `<html>` tag is like opening the notebook. The `<head>` is like the first page where you write your name and class. The `<body>` is where you write all your actual notes.

**🎯 Analogy:**  
An HTML document is like a letter envelope:
- `<!DOCTYPE html>` = The stamp that shows it's official mail
- `<html>` = The envelope itself
- `<head>` = The return address (info about the letter)
- `<body>` = The actual letter inside (the content)
- `</html>` = Sealing the envelope

**🌍 Real World Example:**  
Every single website you visit — Google, YouTube, Amazon, Wikipedia — all follow this exact same HTML document structure.

**💻 Code Syntax:**
```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>My Website</title>
</head>
<body>
  <header>My Website</header>
  <main>
    <h1>Welcome</h1>
    <p>Content goes here.</p>
  </main>
  <footer>Copyright 2026</footer>
</body>
</html>
```

**📜 Rules:**
1. `<!DOCTYPE html>` must be the first line — no spaces before it
2. `<html>` is the root element — everything goes inside it
3. `<head>` comes before `<body>` — this order is mandatory
4. Only one `<head>` and one `<body>` per document
5. The `<title>` tag inside `<head>` is required for valid HTML

#### 🔤 Topic: HTML Headings

| Field | Details |
|-------|---------|
| **Difficulty** | ⭐ Beginner |
| **Estimated Time** | 20 minutes |
| **Prerequisites** | HTML Tags & Elements |

**📖 Definition:**  
HTML headings define the hierarchy of content on a page. There are 6 levels: `<h1>` (most important) to `<h6>` (least important).

**🧠 Roman English Explanation:**  
Headings jaise newspaper ki headlines hoti hain. `<h1>` sabse badi heading hoti hai (main title ke liye), aur `<h6>` sabse chhoti. Search engines headings ko dekhkar samajhte hain ki page mein kya important hai.

**💻 Code Syntax:**
```html
<h1>Main Title (Page ka naam)</h1>
<h2>Section Title (Section ka naam)</h2>
<h3>Sub-section Title</h3>
<h4>Sub-sub-section</h4>
<h5>Small heading</h5>
<h6>Smallest heading</h6>
```

**📜 Rules:**
1. Only ONE `<h1>` per page (for SEO)
2. Headings should follow hierarchy: h1 → h2 → h3 (don't skip levels)
3. Headings are block-level elements (they take full width)
4. Browsers add default margin to headings
5. Don't use headings just for bold text — use CSS instead

| Field | Details |
|-------|---------|
| **Difficulty** | ⭐ Beginner |
| **Estimated Time** | 30 minutes |
| **Prerequisites** | HTML Tags & Elements |

**📖 Definition:**  
The `<p>` tag defines a paragraph. HTML automatically adds margins before and after paragraphs. Text formatting tags change how text looks.

**💻 Code Syntax:**
```html
<p>This is a normal paragraph. Browser will show it as a block of text.</p>

<!-- Text Formatting Tags -->
<b>Bold text</b>
<strong>Important text (also bold)</strong>
<i>Italic text</i>
<em>Emphasized text (also italic)</em>

<s>Strikethrough text</s>
<mark>Highlighted text</mark>
<small>Smaller text</small>
```

**📜 Rules:**

- `<pre>` preserves all whitespace and line breaks exactly as written

#### 🔤 Topic: HTML Links (Anchor Tags)

| Field | Details |
|-------|---------|
| **Difficulty** | ⭐ Beginner |
| **Estimated Time** | 45 minutes |
| **Prerequisites** | HTML Tags & Attributes |

**📖 Definition:**  
The `<a>` (anchor) tag creates hyperlinks to other web pages, files, email addresses, or locations within the same page.

**🧠 Roman English Explanation:**  
Links hi woh cheez hain jo web ko "web" banati hain — interconnected pages. Jab aap kisi link par click karte ho, to aap doosre page par pahunch jaate ho. `<a>` tag ke `href` attribute mein destination URL likha hota hai.

**💻 Code Syntax:**
```html
<!-- Basic link -->
<a href="https://google.com">Visit Google</a>

<!-- Open in new tab -->
<a href="https://google.com" target="_blank" rel="noopener">Google (New Tab)</a>

<!-- Link to same-page section -->
<a href="#section2">Go to Section 2</a>
<h2 id="section2">Section 2</h2>

<!-- Email link -->
<a href="mailto:hello@example.com">Send Email</a>

<!-- Phone link -->
<a href="tel:+1234567890">Call Us</a>

<!-- Download link -->
<a href="file.pdf" download>Download PDF</a>

<!-- Link with title (tooltip) -->
<a href="page.html" title="Go to page">Hover over me</a>

<!-- No-follow link (for SEO) -->
```

#### 🔤 Topic: HTML5 Canvas

| Field | Details |
|-------|---------|
| **Difficulty** | ⭐⭐⭐ Advanced |
| **Estimated Time** | 2 hours |
| **Prerequisites** | HTML Tags, Basic JavaScript |

**📖 Definition:**  
The `<canvas>` element provides a drawable region using JavaScript. It's used for graphics, games, data visualization, and image processing.

**💻 Code Syntax:**
```html
<canvas id="myCanvas" width="500" height="300" style="border:1px solid black;">
  Your browser does not support canvas.
</canvas>

<script>
  const canvas = document.getElementById('myCanvas');
  const ctx = canvas.getContext('2d');

  // Draw rectangle
  ctx.fillStyle = 'red';
  ctx.fillRect(50, 50, 100, 80);

  // Draw circle
  ctx.beginPath();
  ctx.arc(250, 150, 50, 0, Math.PI * 2);
  ctx.fillStyle = 'blue';
  ctx.fill();

  // Draw text
  ctx.font = '30px Arial';
  ctx.fillStyle = 'black';
  ctx.fillText('Hello Canvas!', 50, 250);

  // Draw line
  ctx.beginPath();
  ctx.moveTo(0, 0);
  ctx.lineTo(500, 300);
  ctx.strokeStyle = 'green';
  ctx.stroke();

  // Gradient
  const gradient = ctx.createLinearGradient(0, 0, 500, 0);
  gradient.addColorStop(0, 'red');
  gradient.addColorStop(0.5, 'green');
  gradient.addColorStop(1, 'blue');
  ctx.fillStyle = gradient;
  ctx.fillRect(300, 200, 150, 80);
</script>
```

**📜 Rules:**
1. Canvas dimensions set in HTML attributes (not CSS) for proper rendering
2. Always provide fallback content inside `<canvas>` tags
3. Canvas is pixel-based — once drawn, can't be modified (only redrawn)
4. Animation requires requestAnimationFrame for smooth performance
5. getContext('2d') returns the 2D rendering context

| **Prerequisites** | HTML, JavaScript, DOM |

**📖 Definition:**  
Web Components are a set of browser APIs that allow creating reusable custom HTML elements with encapsulated styles and behavior. They consist of three main technologies: Custom Elements, Shadow DOM, and HTML Templates.

**🧠 Roman English Explanation:**  
Web Components se aap apne khud ke HTML tags bana sakte ho. Jaise `<my-button>`, `<user-card>`, `<rating-star>`. Inke andar ki styling aur logic fully encapsulated hoti hai — bahar se affect nahi ho sakti. Ye teen cheezon se milkar banta hai: Custom Elements (apna tag), Shadow DOM (private styling), aur HTML Templates (reusable structure).

**💻 Code Syntax:**

```html
<!-- Custom Element -->
<script>
  class UserCard extends HTMLElement {
    constructor() {
      super();
      // Shadow DOM
      this.attachShadow({ mode: 'open' });
      this.shadowRoot.innerHTML = `
        <style>
          .card {
            border: 2px solid #333;
            border-radius: 8px;
            padding: 16px;
            margin: 10px;
            font-family: Arial;
          }
          .name { color: blue; font-size: 1.2em; }
        </style>
        <div class="card">
          <div class="name"><slot name="name">Name</slot></div>
          <div><slot name="email">Email</slot></div>
        </div>
      `;
    }
  }
  customElements.define('user-card', UserCard);
</script>

<user-card>
  <span slot="name">Ram Sharma</span>
  <span slot="email">ram@email.com</span>
</user-card>

<!-- HTML Template -->
<template id="myTemplate">
  <style>
    p { color: navy; font-weight: bold; }
  </style>
  <p>This is from a template!</p>
</template>

<script>
  const template = document.getElementById('myTemplate');
  const clone = template.content.cloneNode(true);
  document.body.appendChild(clone);
</script>
```

**📖 Definition:**  
Web accessibility means making websites usable for people with disabilities — including visual, hearing, motor, and cognitive impairments. Accessible HTML is the foundation.

**🧠 Roman English Explanation:**  
Accessibility ka matlab hai ki aapki website har kisi ke liye usable ho — chahe woh blind ho, deaf ho, ya kisi bhi tarah ki disability ho. Screen readers (jo blind log use karte hain) aapke HTML ko padh kar sunate hain. Isliye aapko semantic tags, ARIA attributes, aur proper structure use karna chahiye.

**Key Accessibility Techniques:**

```html
<!-- 1. Use semantic HTML -->
<nav>...</nav>  <!-- Better than <div> -->
<button>Submit</button>  <!-- Better than <div onclick=""> -->

<button aria-label="Close dialog" onclick="closeDialog()">X</button>

<nav aria-label="Main navigation">...</nav>
<div role="tabpanel" aria-labelledby="tab1">...</div>

<!-- 3. Skip to content link -->
<a href="#main-content" class="skip-link">Skip to main content</a>
<main id="main-content">...</main>

<!-- 4. Form accessibility -->
<label for="email">Email address:</label>
<input type="email" id="email" name="email" aria-describedby="emailHelp" aria-required="true">
<span id="emailHelp">We'll never share your email.</span>
<span role="alert" id="email-error" hidden>Please enter a valid email</span>

<!-- 5. Image alt text -->
<img src="chart.png" alt="Bar chart showing sales growth from 100 to 500 in 2025">
<!-- For decorative images -->
<img src="decorative-border.png" alt="" role="presentation">

<!-- 6. Keyboard navigation -->
<button onkeydown="if(event.key==='Enter') this.click()">Click</button>

<!-- 7. Focus management -->
<div tabindex="0">This can receive focus</div>
<div tabindex="-1">This can programmatically receive focus</div>
```

**📜 WCAG Key Principles (POUR):**

| Principle | Meaning | Example |
|-----------|---------|---------|
| **P**erceivable | Content must be perceivable by all senses | Alt text for images, captions for videos |
| **O**perable | Interface must be operable | Keyboard navigation, enough time |
| **U**nderstandable | Content and interface must be understandable | Clear labels, predictable behavior |
| **R**obust | Content must work with current/future tools | Valid HTML, proper ARIA usage |

#### 🔤 Topic: What is CSS?

| Field | Details |
|-------|---------|
| **Difficulty** | ⭐ Beginner |
| **Estimated Time** | 30 minutes |
| **Prerequisites** | HTML Basics |
| **Learning Objectives** | Understand what CSS is, its purpose, and how it styles web pages |

**📖 Definition:**  
CSS (Cascading Style Sheets) is a style sheet language used to describe the presentation of HTML documents. It controls colors, fonts, layouts, animations, and responsive design.

**🧠 Roman English Explanation:**  
CSS woh language hai jo aapke HTML page ko sundar banati hai. Jaise aap kisi ghar ka skeleton (HTML) banane ke baad usme paint, furniture, decoration (CSS) karte ho. CSS ke bina webpages sirf black text white background par dikhte — boring! CSS se aap colors, fonts, layout, animation sab kuch control kar sakte ho.

**🔬 Technical Explanation:**  
CSS works by selecting HTML elements and applying style rules to them. A CSS rule consists of a selector (what to style) and a declaration block (how to style it). CSS follows the cascade principle — multiple rules can apply to the same element, and specificity, inheritance, and source order determine which rule wins.

**👶 Simple Explanation:**  
CSS is like the clothes and makeup for your HTML skeleton. HTML decides what goes where, CSS decides how it looks — what color, how big, where exactly, and how it moves.

**🎯 Analogy:**  
Imagine building a car:
- HTML = The chassis (body structure)
- CSS = The paint, seats, dashboard, rims (appearance)
- JavaScript = The engine, steering, brakes (behavior)

**📜 History & Evolution:**
- **1996:** CSS1 released (basic styling)
- **1998:** CSS2 released (positioning, media types)
- **2011:** CSS3 became modular (no single version — individual modules at different levels)
- **2026:** CSS is continuously evolving with modules like Container Queries, Cascade Layers, View Transitions API

**💻 Code Syntax:**
```css
/* Basic CSS Rule */
selector {
  property: value;
  property: value;
}

/* Example */
h1 {
  color: blue;
  font-size: 24px;
  text-align: center;
}

p {
  color: gray;
  line-height: 1.6;
}
```

**📜 Rules:**
1. CSS rules end with semicolons (`;`)
2. Properties and values are case-sensitive
3. Comments use `/* comment */` syntax
4. Last rule wins in case of conflict (if same specificity)

#### 🔤 Topic: CSS Box Model

| Field | Details |
|-------|---------|
| **Difficulty** | ⭐⭐ Intermediate |
| **Estimated Time** | 1 hour |
| **Prerequisites** | CSS Selectors, Units |

**📖 Definition:**  
The CSS Box Model is the foundation of layout. Every HTML element is a rectangular box with four layers: content, padding, border, and margin.

**🧠 Roman English Explanation:**  
Har HTML element ek box ki tarah hota hai. Is box ke chaar layers hote hain. Andar sabse content hota hai (text ya image). Uske around padding hota hai (element ke andar ka space). Phir border hota hai (element ki boundary). Aur sabse bahar margin hota hai (doosre elements se distance).

**📊 Box Model Diagram:**
```
┌─────────────────────────────────────┐
│              MARGIN (outer)          │
│  ┌───────────────────────────────┐  │
│  │          BORDER              │  │
│  │  ┌─────────────────────────┐  │  │
│  │  │       PADDING          │  │  │
│  │  │  ┌───────────────────┐  │  │  │
│  │  │  │                   │  │  │  │
│  │  │  │     CONTENT       │  │  │  │
│  │  │  │                   │  │  │  │
│  │  │  └───────────────────┘  │  │  │
│  │  └─────────────────────────┘  │  │
│  └───────────────────────────────┘  │
└─────────────────────────────────────┘
```

**💻 Code Syntax:**
```css
.box {
  /* Content */
  width: 300px;
  height: 200px;

  /* Padding (inside, between content and border) */
  padding-top: 20px;
  padding-right: 20px;
  padding-bottom: 20px;
  padding-left: 20px;
  /* Shorthand: padding: top right bottom left */
  padding: 20px;  /* all 4 sides */
  padding: 10px 20px;  /* top/bottom left/right */
  padding: 10px 20px 15px; /* top left/right bottom */
  padding: 10px 20px 15px 5px; /* top right bottom left */

  /* Border (edge of the box) */
  border-width: 2px;
  border-style: solid; /* solid, dashed, dotted, double, groove, ridge, inset, outset, none */
  border-color: black;
  /* Shorthand */
  border: 2px solid black;
  border-top: 2px dashed red;
  border-radius: 8px; /* rounded corners */

  /* Margin (outside, between this box and others) */
  margin-top: 10px;
  margin-right: 10px;
  margin-bottom: 10px;
  margin-left: 10px;
  /* Shorthand */
  margin: 10px;
  margin: 10px 20px;
  margin: 10px auto; /* center block elements horizontally */
}

/* Box Sizing - IMPORTANT! */
* {
  box-sizing: border-box;  /* padding and border INSIDE the width */
  /* Default: box-sizing: content-box (padding/border ADD to width) */
}

/* Difference */
.content-box {
  box-sizing: content-box;
  width: 300px;
  padding: 20px;
  border: 5px solid black;
  /* Total width = 300 + 20 + 20 + 5 + 5 = 350px */
}

.border-box {
  box-sizing: border-box;
  width: 300px;
  padding: 20px;
  border: 5px solid black;
  /* Total width = 300px (content width = 300 - 40 - 10 = 250px) */
}
```

**📜 Box Model Rules:**
1. Block elements take full width by default, inline elements don't respect width/height
2. Vertical margins collapse (larger margin wins, not sum)
3. Use `box-sizing: border-box` globally for predictable layouts
4. `margin: auto` centers block elements horizontally
5. Padding adds INSIDE the element, margin adds OUTSIDE

#### 🔤 Topic: CSS Position Property

| Field | Details |
|-------|---------|
| **Difficulty** | ⭐⭐⭐ Advanced |

**Complete Positioning Reference:**

| Value | Behavior | Offset Properties |
|-------|----------|-------------------|
| `static` | Default, normal flow | Not applicable |
| `relative` | Relative to normal position | top, right, bottom, left |
| `absolute` | Relative to nearest positioned ancestor | top, right, bottom, left |
| `fixed` | Relative to viewport | top, right, bottom, left |
| `sticky` | Toggle between relative/fixed based on scroll | top, right, bottom, left |

```css
/* Static (default) */
.static { position: static; }

/* Relative - stays in flow, offset from normal position */
.relative {
  position: relative;
  top: 20px;
  left: 30px;
}

/* Absolute - removed from flow, positioned relative to nearest positioned ancestor */
.container { position: relative; }  /* reference point */
.absolute-child {
  position: absolute;
  top: 0;
  right: 0;
}

/* Fixed - relative to viewport, stays on scroll */
.fixed-header {
  position: fixed;
  top: 0;
  left: 0;
  width: 100%;
  z-index: 1000;
}

/* Sticky - relative until scroll threshold, then fixed */
.sticky-nav {
  position: sticky;
  top: 0;
  z-index: 100;
}

/* Z-index (stacking order) */
.layer1 { z-index: 1; }    /* Lowest */
.layer2 { z-index: 10; }   /* Higher */
.layer3 { z-index: 999; }  /* Highest */
```

3. `sticky` requires a `top`/`bottom` threshold value to work
4. `z-index` only works on positioned elements (not static)
5. Higher `z-index` = appears on top

#### 🔤 Topic: CSS Flexbox (Complete Guide)

| Field | Details |
|-------|---------|
| **Difficulty** | ⭐⭐⭐ Advanced |

| **Prerequisites** | Display, Position |

**📖 Definition:**  
Flexbox (Flexible Box Layout) is a 1D layout model that distributes space and aligns items in a container. It works in one direction — either row (horizontal) or column (vertical).

**🧠 Roman English Explanation:**  
Flexbox ek modern layout system hai jo elements ko ek line mein arrange karta hai — ya to horizontally (row) ya vertically (column). Isse aap easily center kar sakte ho, space distribute kar sakte ho, aur responsive layouts bana sakte ho bina floats ya tables ke.

**Flexbox Architecture:**
```
Flex Container
┌──────────────────────────────────────────────┐
│  Main Axis (horizontal by default)           │
│  ──────────────────────────────────────────►  │
│                                                │
│  ┌──────┐  ┌──────┐  ┌──────┐  ┌──────┐     │
│  │Item 1│  │Item 2│  │Item 3│  │Item 4│     │
│  └──────┘  └──────┘  └──────┘  └──────┘     │
│  ▲                                           │
│  │ Cross Axis                                │
│  │ (vertical)                                │
│                                                │
└──────────────────────────────────────────────┘
```

**Complete Flexbox Properties Reference:**

**Container Properties:**
```css
.flex-container {
  /* Enable flexbox */
  display: flex;          /* block-level flex container */
  display: inline-flex;   /* inline-level flex container */

  /* Direction */
  flex-direction: row;             /* default: left to right */
  flex-direction: row-reverse;     /* right to left */
  flex-direction: column;          /* top to bottom */
  flex-direction: column-reverse;  /* bottom to top */

  /* Wrap */
  flex-wrap: nowrap;      /* default: no wrapping */
  flex-wrap: wrap;        /* items wrap to next line */
  flex-wrap: wrap-reverse; /* wrap in opposite direction */

  /* Shorthand: flex-flow: [direction] [wrap] */
  flex-flow: row wrap;

  /* Main Axis Alignment (horizontal if row) */
  justify-content: flex-start;    /* default: items at start */
  justify-content: flex-end;      /* items at end */
  justify-content: center;        /* items centered */
  justify-content: space-between; /* equal space between, none at ends */
  justify-content: space-around;  /* equal space around each item */
  justify-content: space-evenly;  /* perfectly equal space */

  /* Cross Axis Alignment (vertical if row) */
  align-items: stretch;     /* default: fill container height */
  align-items: flex-start;  /* items at top */
  align-items: flex-end;    /* items at bottom */
  align-items: center;      /* items centered */
  align-items: baseline;    /* align by text baseline */

  /* Multi-line Alignment */
  align-content: stretch;     /* default */
  align-content: flex-start;  /* lines at start */
  align-content: flex-end;    /* lines at end */
  align-content: center;      /* lines centered */
  align-content: space-between;
  align-content: space-around;
  align-content: space-evenly;

  /* Gap (spacing between items) */
  gap: 16px;           /* shorthand */
  row-gap: 16px;       /* vertical gap */
  column-gap: 16px;    /* horizontal gap */
}
```

**Item Properties:**
```css
.flex-item {
  /* Order (rearrange items) */
  order: 0;      /* default: 0, can be negative or positive */

  /* Grow (take extra space) */
  flex-grow: 0;     /* default: don't grow */
  flex-grow: 1;     /* take remaining space equally */

  /* Shrink (reduce when not enough space) */
  flex-shrink: 1;   /* default: can shrink */
  flex-shrink: 0;   /* never shrink */

  /* Basis (initial size before growing/shrinking) */
  flex-basis: auto;  /* default: based on content */
  flex-basis: 200px; /* start at 200px */

  /* Shorthand: flex: [grow] [shrink] [basis] */
  flex: 1;                      /* flex: 1 1 0% */
  flex: 1 0 auto;               /* grow but don't shrink */
  flex: 0 0 200px;              /* fixed 200px (no grow, no shrink) */

  /* Individual alignment (overrides align-items) */
  align-self: auto;        /* inherits from container */
  align-self: flex-start;  /* top */
  align-self: flex-end;    /* bottom */
  align-self: center;      /* middle */
  align-self: stretch;     /* full height */
}
```

**Common Flexbox Patterns:**

```css
/* 1. Perfect Centering */
.center {
  display: flex;
  justify-content: center;
  align-items: center;
}

/* 2. Equal-width columns */
.equal-columns {
  display: flex;
}

  flex: 1;
}

/* 3. Sticky Footer */

  display: flex;
  flex-direction: column;
  min-height: 100vh;

  flex: 1; /* takes all remaining space */
}

  /* stays at bottom */
}

/* 4. Holy Grail Layout */
.holy-grail {
  display: flex;
  min-height: 100vh;
  flex-direction: column;
}
.holy-grail-body {
  display: flex;
  flex: 1;
}
.holy-grail-content {
  flex: 1;
}
.holy-grail-nav, .holy-grail-ads {
  flex: 0 0 200px;
}

/* 5. Responsive cards */

  display: flex;
  flex-wrap: wrap;
  gap: 20px;
```

CSS Grid Layout is a 2D layout system that can handle both columns and rows simultaneously. It's the most powerful layout system in CSS.

Agar Flexbox 1D hai (sirf row ya column), to Grid 2D hai (row aur column dono ek saath). Grid se aap table jaisa complex layout bana sakte ho, lekin zyada flexible aur responsive. Aap columns define karte ho, rows define karte ho, aur items ko grid ke andar rakh dete ho.

**📖 Definition:**  
Responsive design makes websites look good on all devices — phones, tablets, laptops, desktops. Media queries are CSS rules that apply styles based on device characteristics like width, height, resolution, and orientation.

**🧠 Roman English Explanation:**  
Aaj kal log phone, tablet, laptop, desktop — sab par websites dekhte hain. Responsive design ka matlab hai ki aapki website har screen size par acchi lage. Media queries CSS mein conditions ki tarah hote hain — "agar screen 768px se chhoti hai to ye styling lagao, agar 1200px se badi hai to ye styling lagao."

```css
/* Mobile-First Approach (recommended) */

/* Base styles (mobile first - no media query) */
.container {
  display: grid;
  grid-template-columns: 1fr;
  gap: 16px;
  padding: 16px;
}

/* Tablet (768px+) */
@media (min-width: 768px) {
  .container {
    grid-template-columns: repeat(2, 1fr);
    padding: 24px;
  }
}

/* Desktop (1024px+) */
@media (min-width: 1024px) {
  .container {
    grid-template-columns: repeat(3, 1fr);
    max-width: 1200px;
    margin: 0 auto;
  }
}

/* Large Desktop (1440px+) */
@media (min-width: 1440px) {
  .container {
    grid-template-columns: repeat(4, 1fr);
    gap: 32px;
  }
}

/* Other Media Query Types */
/* Print styles */
@media print {
  nav, .ads, .sidebar { display: none; }
  body { font-size: 12pt; }
}

/* Dark mode */
@media (prefers-color-scheme: dark) {
  body {
    background: #121212;
    color: #ffffff;
  }
}

/* Reduced motion (accessibility) */
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}

/* Portrait vs Landscape */
@media (orientation: portrait) { ... }
@media (orientation: landscape) { ... }

/* High DPI (retina) screens */
@media (-webkit-min-device-pixel-ratio: 2), (min-resolution: 192dpi) {
  /* Use 2x images */
}

/* Container queries (modern) */
@container (min-width: 400px) {
  .card { display: flex; }
}
```

#### 🔤 Topic: JavaScript Variables (var, let, const)

| Field | Details |
|-------|---------|
| **Difficulty** | ⭐ Beginner |
| **Estimated Time** | 1 hour |
| **Prerequisites** | What is JS |

**📖 Definition:**  
Variables are containers that store data values. JavaScript has three ways to declare variables: `var`, `let`, and `const`.

**🧠 Roman English Explanation:**  
Variables aise boxes hain jisme aap data rakh sakte ho. Jaise `age = 25` — yahan `age` variable hai jisme value 25 store hai. `let` aur `const` modern hain, `var` purana hai. `const` woh variable jise aap change nahi kar sakte (fixed), `let` ko change kar sakte ho.

```javascript
// ============ var (old way, avoid using) ============
var name = 'Ram';     // function-scoped
var name = 'Shyam';   // can be redeclared (problem!)
console.log(name);    // Shyam

// ============ let (modern, mutable) ============
let age = 25;
age = 26;             // can reassign
// let age = 30;      // ERROR: cannot redeclare in same scope

// ============ const (modern, immutable reference) ============
const pi = 3.14159;
// pi = 3;            // ERROR: cannot reassign
const person = { name: 'Ram' };
person.name = 'Shyam'; // OK: object properties can change
// person = {};       // ERROR: cannot reassign entire object

// ============ Scope Differences ============
// var is function-scoped
function example() {
  if (true) {
    var x = 10;   // accessible outside if block
  }
  console.log(x); // 10
}

// let and const are block-scoped
function example2() {
  if (true) {
    let y = 10;   // only inside if block
    const z = 20; // only inside if block
  }
  // console.log(y); // ReferenceError
}

// ============ Hoisting ============
console.log(a);  // undefined (var hoisted but not initialized)
var a = 5;

// console.log(b); // ReferenceError (let hoisted but in "temporal dead zone")
let b = 5;

// ============ Naming Conventions ============
let firstName = 'Ram';      // camelCase (standard for JS)
let FIRST_NAME = 'RAM';     // UPPER_SNAKE_CASE (constants)
let isActive = true;        // boolean prefix (is, has, should)
let userList = [];          // plural for arrays
```

**📜 When to use what:**
- **`const`** by default (unless you need to reassign)
- **`let`** when you need to reassign (loop counters, accumulators)
- **`var`** NEVER in modern JS (legacy code only)

```javascript
// ============ 1. ARITHMETIC OPERATORS ============
let a = 10, b = 3;
console.log(a + b);   // 13  (addition)
console.log(a - b);   // 7   (subtraction)
console.log(a * b);   // 30  (multiplication)
console.log(a / b);   // 3.333... (division)
console.log(a % b);   // 1   (modulus/remainder)
console.log(a ** b);  // 1000 (exponentiation, ES7)

// Increment/Decrement
let x = 5;
console.log(x++); // 5 (post-increment: returns then adds)
console.log(x);   // 6
console.log(++x); // 7 (pre-increment: adds then returns)

// ============ 2. ASSIGNMENT OPERATORS ============
let y = 10;
y += 5;   // y = y + 5
y -= 3;   // y = y - 3
y *= 2;   // y = y * 2
y /= 4;   // y = y / 4
y %= 3;   // y = y % 3
y **= 2;  // y = y ** 2
y &&= 5;  // logical AND assignment (ES2021)
y ||= 10; // logical OR assignment
y ??= 20; // nullish coalescing assignment

// ============ 3. COMPARISON OPERATORS ============
console.log(5 == '5');   // true  (loose equality - type coercion)
console.log(5 === '5');  // false (strict equality - no coercion)
console.log(5 != '5');   // false (loose inequality)
console.log(5 !== '5');  // true  (strict inequality)
console.log(5 > 3);      // true
console.log(5 < 3);      // false
console.log(5 >= 5);     // true
console.log(5 <= 4);     // false

// ============ 4. LOGICAL OPERATORS ============
console.log(true && false);  // false (AND)
console.log(true || false);  // true  (OR)
console.log(!true);          // false (NOT)

// Short-circuit evaluation
let result = false && console.log('never runs'); // doesn't run
let result2 = true || console.log('never runs'); // doesn't run

// ============ 5. TERNARY OPERATOR ============
let age = 20;
let status = age >= 18 ? 'Adult' : 'Minor';
console.log(status); // 'Adult'

// ============ 6. NULLISH COALESCING (ES2020) ============
let userInput = null;
let value = userInput ?? 'default'; // 'default'
// ?? vs ||: ?? only checks null/undefined, || checks all falsy
let count = 0;
console.log(count || 10);  // 10 (0 is falsy)
console.log(count ?? 10);  // 0  (0 is not null/undefined)

// ============ 7. OPTIONAL CHAINING (ES2020) ============
let user = { address: { city: 'Delhi' } };
console.log(user?.address?.city);     // 'Delhi'
console.log(user?.contact?.phone);    // undefined (no error!)
console.log(user?.greet?.());         // undefined (method may not exist)
console.log(arr?.[0]);                // array element access

// ============ 8. SPREAD/REST OPERATORS (ES6) ============
// Spread (expand)
let arr1 = [1, 2, 3];
let arr2 = [...arr1, 4, 5];  // [1, 2, 3, 4, 5]
let obj1 = { a: 1, b: 2 };
let obj2 = { ...obj1, c: 3 }; // { a: 1, b: 2, c: 3 }

// Rest (collect)
function sum(...numbers) { return numbers.reduce((a, b) => a + b); }
console.log(sum(1, 2, 3, 4)); // 10

// ============ 9. DESTRUCTURING (ES6) ============
// Array destructuring
let [first, second, ...rest] = [1, 2, 3, 4, 5];
console.log(first, second, rest); // 1, 2, [3, 4, 5]

// Object destructuring
let { name, age: personAge } = { name: 'Ram', age: 25 };
console.log(name, personAge); // 'Ram', 25

// ============ 10. typeof & instanceof ============
console.log(typeof 'hello');  // 'string'
console.log([] instanceof Array); // true
```

```javascript
// An object representing the eventual completion/failure of an async operation
// States: pending → fulfilled OR rejected

// ============ 2. CREATING PROMISES ============
const myPromise = new Promise((resolve, reject) => {
  // Async operation
  setTimeout(() => {
    const success = true;
    if (success) {
      resolve('Operation successful!');
    } else {
      reject(new Error('Operation failed'));
    }
  }, 1000);
});

// ============ 3. CONSUMING PROMISES ============
myPromise
  .then((result) => {
    console.log('Success:', result);
    return 'Next value'; // chaining
  })
  .then((nextValue) => {
    console.log('Chained:', nextValue);
  })
  .catch((error) => {
    console.error('Error:', error.message);
  })
  .finally(() => {
    console.log('Always runs');
  });

// ============ 4. PROMISE METHODS ============

  .catch(err => console.log('One failed:', err));
```

```jsx
import { useState, useEffect } from 'react';

function useLocalStorage(key, initialValue) {
  const [storedValue, setStoredValue] = useState(() => {
    try {
      const item = window.localStorage.getItem(key);
      return item ? JSON.parse(item) : initialValue;
    } catch (error) {
      return initialValue;
    }
  });

  useEffect(() => {
    try {
      window.localStorage.setItem(key, JSON.stringify(storedValue));
    } catch (error) {
      console.error(error);
    }
  }, [key, storedValue]);

  return [storedValue, setStoredValue];
}

class ErrorBoundary extends React.Component {
  constructor(props) {
    super(props);
    this.state = { hasError: false, error: null };

  static getDerivedStateFromError(error) {
    return { hasError: true, error };
  }

  componentDidCatch(error, errorInfo) {
    console.error('Error caught:', error, errorInfo);
  }

  render() {
```

```sql
INNER JOIN enrollments e ON s.id = e.student_id
INNER JOIN courses c ON e.course_id = c.id
WHERE s.age >= 18 AND s.is_active = true
GROUP BY s.id
HAVING COUNT(e.id) > 2
ORDER BY s.name ASC
LIMIT 10 OFFSET 0;

FROM students
WHERE id IN (SELECT student_id FROM enrollments WHERE course_id = 101);

FROM courses c
LEFT JOIN enrollments e ON c.id = e.course_id
LEFT JOIN students s ON e.student_id = s.id
GROUP BY c.id
ORDER BY enrolled_students DESC;
```
